// Why trust the output

Not just generated. Proven.

Every scaffold leaves a paper trail: what was requested, which conventions and providers ran, and whether verification actually passed.

01
Full prompt shown first
Forge displays the complete prompt before execution and uses bounded provider workspaces by default.
02
Doctor without a model call
forge doctor checks installation and authentication deterministically — no credentials printed, no quota used.
03
Deterministic conventions
Convention layers are recorded with source hashes, so every scaffold is replayable and auditable.
04
Safe by default
Blanket provider bypass requires two explicit unsafe flags. Provider-default models stay in control unless you override.
05
Resume, don’t rerun
Atomic progress and failure classification let --resume pick up where a run stopped, without paying twice.
06
Verified means verified
.forge/ records distinguish “provider finished” from “generated project passed its own checks.”

// Install

Up and running in one line

Then forge doctor confirms your provider is ready — no model call, no credentials printed.

Homebrew Recommended
The supported tap for macOS and Linux. Installs the projectforge command with pinned dependencies.
brew install schramm2/tap/projectforge
uv / pipx
Latest published release from PyPI, installed as an isolated tool. Also works with pipx install.
uv tool install matt-projectforge
From source
Track unreleased changes after v0.6.0. Requires uv and Python 3.12+.
git clone https://github.com/Schramm2/projectforge.git
cd projectforge && uv sync --dev
./forge --version
Ethereum's forge already on your PATH? Use the projectforge command — forge stays as a compatibility alias.

The dashboard reports Project Ready only when required verification passes. Skipped checks give the honest Project Created instead.

verification.json Redacted commands, timeouts, exits, endpoints, remediation
